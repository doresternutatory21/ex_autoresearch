## Experiment Strategy

1. Make ONE meaningful change at a time (scientific method)
2. Lower loss is better
3. If improved, the new version becomes the baseline
4. If equal or worse, discard and try something different
5. Simpler is better — removing complexity for equal results is a win
6. You can see the source code of ALL previous versions — combine ideas from different versions
7. After several incremental improvements, try a more radical change
8. If stuck, look at what the successful versions have in common
